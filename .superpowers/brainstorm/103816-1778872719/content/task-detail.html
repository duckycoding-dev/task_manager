Recurrence picker (single field, derived from backend's two fields)

Backend reality (two redundant fields)

isRecurring:       boolean
recurringInterval: 'daily' | 'weekly' |
                   'monthly' | 'none'

The two fields can disagree (isRecurring=true, recurringInterval='none' is technically allowed by the schema). v1 frontend collapses them into one picker; backend writes the consistent pair.

User-facing picker

✓Weekly every 7 days from due date
Daily
Monthly
Never single occurrence

Mapping: Never → { isRecurring:false, recurringInterval:'none' }. Any other option → { isRecurring:true, recurringInterval:<choice> }.
