Spec notes

  • Settings IA — Sub-nav routes: Profile · Display & layout · Notifications · Data · About. Mobile: sub-nav collapses to a list page (`/settings`); each item navigates to its own route on tap. Back button returns to the list.
  • Preference persistence — All Display & layout settings stored under users/preferences (v1: localStorage; v1.5: backend endpoint adds sync). Settings page writes to whichever is current.
  • Profile — BetterAuth endpoints already handle email change + password change. Email change kicks off verification (already in schema). Display name = part of user record.
  • Notifications section — v1 placeholder. Surfaces: email digest, reminder fire mechanism. 🟡 backend gap — no email/push delivery today. Section is visible but most controls are disabled with a "Coming soon" note. Or hide the section entirely for v1 — call it.
  • Data section — Export tasks (CSV/JSON), import (later), delete-all-data (different from delete-account; nukes the data but keeps the account). 🟡 backend export endpoint needed.
  • Auth screens — Standalone shell (no top bar / sidebar / bottom nav). Centered card on a subtle gradient background (the same violet/cyan accents from the palette, very low-saturation). Cards: --shadow-pop + --radius-xl. Mobile: card becomes full-width with margin.
  • Auth routes — /auth/login, /auth/signup, /auth/forgot, /auth/reset?token=…, /auth/verify?token=…. All standalone.
  • Post-auth redirect — Sign-in success → Dashboard (or the originally-requested route if user was bounced from a protected page).
