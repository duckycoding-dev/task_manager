Per-row actions explained

  • Now (fired, awaiting action) · Done · Snooze ▾ (15m / 1h / 3h / tomorrow / pick…) · × Dismiss (soft-delete). Snooze updates remindAt to a future time.
  • Upcoming · Edit (opens reminder dedicated page) · × Delete (soft-delete).
  • Past · read-only, shown only when expanded. Click → reminder page (with restore option). v1 default = collapse, show only count.

Backend reality / gaps

🟡 No "fired / acknowledged" state on reminders. Today's reminders schema is just { id, taskId, userId, title, content, remindAt }. There's no acknowledgedAt or doneAt. The "Now · awaiting action" bucket in v1 is **derived**: remindAt <= now AND deletedAt IS NULL. "Done" and "Dismiss" both perform a soft delete (deletedAt = now). Distinguishing the two would need acknowledgedAt — deferred to v1.5 (paired with "Past · done" filter).
🟡 Snooze is a PATCH /reminders/:id writing a new remindAt. Already supported. No new endpoint needed; preset options (15m / 1h / 3h / tmrw / pick…) are pure client-side.
🟢 Soft delete on reminders. Already locked from earlier sections; "Dismiss × " uses it.

Future expansion · notifications surface

v2+ might add system notifications (mentions, comment replies, due-date approaching warnings, etc.). Those would slot in as an additional section at the top of the page, or as a sibling tab. Schema would be a new notifications table. v1 ships reminder-only and the page name "Inbox" leaves room for that growth.
