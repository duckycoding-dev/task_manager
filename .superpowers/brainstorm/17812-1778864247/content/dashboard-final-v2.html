Quick-add flow (silent defaults)

1
User types title, hits ⏎
Field re-focuses for the next entry (rapid-capture mode).
2
POST /tasks with { title, dueDate: today, status: 'todo', priority: 'medium', isRecurring: false, recurringInterval: 'none', projectId: null }
All omitted-but-required values use backend defaults already declared in tasks.db.ts. Only dueDate=today is a client-side override (so it shows up on this page).
3
New task row appears in "Today", briefly pulse-highlighted
"click to edit" hint visible for ~3s, then fades. Click the row → opens detail panel (route adds ?task=<id>).
4
In the detail panel: set project, priority, labels, description, reminders, recurring
Same panel used everywhere — single source of truth for "all the fields a task has".

Why this works

  • No new picker components needed for v1 (date / project / label / priority / recur pickers all live in the detail panel, which gets one shared implementation).
  • Capture is one input + Enter. Fastest possible.
  • The user still gets a full edit-affordance, just one click later.
  • FAB on mobile + ⌘N anywhere = open the same detail panel pre-focused on title (or a bottom-sheet form on mobile). Decided in task-detail spec.
