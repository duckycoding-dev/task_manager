Tabs detail

  • Board · drag-to-status kanban, 3 fixed columns from backend enum. Click card → detail panel.
  • Tasks · reuses the All-tasks list component with projectId hardcoded. All-tasks filters/columns/density/grouping behavior. Header "+ New task" prefills projectId.
  • Reminders · list of all reminders linked to tasks in this project, sorted by remindAt. Same row pattern as the Inbox page (drafted later). Each shows the parent task name + the link.

Notes on backend reality

  • Status enum is fixed at todo / in_progress / done — board columns are hardcoded to match. v1 does not support custom columns per project.
  • Project color field becomes load-bearing here (cards, board header dot). Already locked as a planned backend addition.
  • "+ Add task" inside a board column sends POST /tasks with { status: 'todo' | 'in_progress' | 'done', projectId: thisProject, title } — same payload shape as quick-add, just with status + projectId prefilled.
  • Drag-to-status sends PATCH /tasks/:taskId/status. Optimistic UI; rollback + toast on failure.
